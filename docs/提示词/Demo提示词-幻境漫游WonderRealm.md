开发一个名为"幻境漫游 WonderRealm"的2D等距视角中式奇幻社交世界Demo，这是一个可以"走进去"的多人社交平行生活空间。以下是完整的Demo需求：

---

## 一、产品概述

一款2D等距俯视视角（isometric）的多人社交互动应用，采用类似《动物森友会》的等距视角美术风格，融合中式文化美学与奇幻元素。核心体验：用户进入一个奇幻世界，看到其他在线玩家在其中活动，自由探索美丽风景、与玩家聊天互动、参加文化社交活动（诗词比赛、婚礼宴请、节日庆典）、在虚拟集市购买服饰装扮自己的角色。

**社交是核心卖点**：场景中始终有其他玩家在场——单人打开时有模拟玩家自主活动，多人打开时能实时看到对方移动和聊天。

技术要求：单个自包含HTML文件，所有CSS和JS内联，使用Canvas渲染2D等距视角。内置一个极简的WebSocket同步模块（用Node.js运行，约80行代码），实现两个浏览器之间的实时位置同步和文字聊天。单浏览器模式下自动启用模拟玩家填充世界。

## 二、必须实现的核心功能（完成度——确保核心链路完整）

按以下顺序实现完整用户旅程，每一步都必须可操作、有反馈，不能有任何断链：

### 2.1 角色创建页（入口页面）
- 提供一个精美的角色创建界面，用户可以选择：3种预设角色形象（手绘Q版中式风格，用不同颜色区分：青衫书生/粉衣少女/白衣剑客）
- 可选择角色名称（输入框，默认"旅人"）
- 底部显示一行说明："打开多个浏览器窗口可与其他玩家实时互动"
- 点击"进入世界"按钮后，过渡动画进入主场景

### 2.2 主场景：云端仙山（等距视角可探索地图 + 多人在线）

#### 场景美术
- 使用2D等距视角（isometric）渲染一个中式奇幻场景，包含：
  - 云海中悬浮的仙山（2-3层叠的等距山体，用渐变色和云雾效果）
  - 山间的中式亭台楼阁（等距视角的简笔建筑，飞檐翘角造型）
  - 瀑布从悬崖倾泻（用粒子或Canvas动画模拟流水效果）
  - 海棠树若干棵，有花瓣飘落动画
  - 石板小路连接各个区域
  - 云雾飘动效果（Canvas粒子或CSS动画）
- 昼夜循环：天空颜色缓慢变化（暖橙→深蓝→粉紫→暖橙，循环），大约60秒一个完整周期

#### 玩家角色操控
- 角色可以用键盘WASD或方向键在地图上8方向移动（等距视角的移动映射），也可以点击地面移动（自动寻路到点击位置）
- 角色移动时有简单的走路动画（2帧切换即可）
- 角色头顶始终显示名称标签

#### 模拟玩家系统（核心社交感来源）
场景中始终存在3-5个"模拟玩家"（当没有真实玩家连接时），让世界看起来"有人气"：
- 每个模拟玩家有随机的中式风格名字（如"云游书生"、"青莲居士"、"月下花影"等，预设10个名字随机取）
- 模拟玩家有不同的服装颜色，与玩家角色视觉上有区分
- 模拟玩家会自主在地图上走动（简单的AI：随机选择目标点→走过去→停留2-5秒→再选新目标点）
- 模拟玩家走路时也有2帧动画
- 模拟玩家偶尔会在头顶弹出聊天气泡，显示预设的中式风格聊天内容（如"今日仙山云海真美"、"有人一起品茶吗"、"刚在集市买了把新折扇"等，预设15条聊天内容随机触发）
- 模拟玩家之间也会有"社交距离"——如果两个模拟玩家走到附近，会触发一个挥手表情动作

#### 真实玩家同步（多人模式）
- 当用户打开第二个浏览器窗口进入同一场景时，两个真实玩家的位置、移动、名称实时同步
- 玩家A移动时，玩家B的屏幕上能看到玩家A在走动
- 玩家可以按Enter键或点击聊天输入框发送文字消息，消息会以气泡形式显示在发送者头顶，所有其他玩家可见
- 模拟玩家在检测到真实玩家连接后自动减少（每多一个真实玩家减少一个模拟玩家，最少保留1个）
- 玩家离开（关闭浏览器）后，该玩家从其他人的场景中消失，模拟玩家数量恢复

#### 场景内NPC
- 场景中散布2-3个可交互的NPC（与玩家颜色/风格有区分，头顶有"！"标记），点击NPC可以触发一段对话（气泡对话框显示2-3句中文对话后消失）
- NPC对话内容包含引导信息，如"点击下方'活动'可以参加诗词雅集"、"集市上新到了一批汉服"等

### 2.3 场景切换：底部导航栏
- 底部固定一个导航栏，包含4个Tab按钮（带图标和文字）：
  - "世界"（当前风景探索页面）
  - "活动"（文化活动页面）
  - "集市"（虚拟商店页面）
  - "我"（个人背包/装扮页面）
- 切换Tab时有平滑过渡动画
- 当前选中的Tab高亮显示
- 导航栏右侧显示一个在线人数指示器（如"在线 5人"，包含真实玩家+模拟玩家）

### 2.4 活动页：诗词比赛（多人参与感）

#### 活动大厅
- 活动页顶部显示"当前活动"区域，以卡片形式展示可参与的活动：
  - "诗词飞花令"卡片（高亮"进行中"标签）
  - "婚礼宴请"卡片（显示"可预约"标签）
- 每张卡片显示一个"参与人数"（如"12人参与"），包含模拟数据，传递"有其他人在参与"的社交感

#### 诗词飞花令
- 点击"诗词飞花令"卡片进入小游戏界面：
  - 顶部显示当前关键字（如"月"字），用中式书法风格大字呈现
  - 中间显示一个古典卷轴样式的诗词展示区
  - 卷轴左侧显示一个"参与者"头像列表（3-5个小头像，用不同颜色圆圈表示其他玩家，含模拟玩家），其中一个高亮标记为"你"
  - 底部显示4个选项按钮（含有一句含"月"字的正确诗句和3个干扰项）
  - 选择正确后显示"答对了！"绿色反馈动画+得分+10
  - 选择错误显示"再想想~"红色提示
  - 每次答题后，参与者列表中其他"玩家"的分数也会变化（模拟其他人在同时答题），营造"实时竞技"的社交感
  - 连续答对3题后显示"恭喜通关！"庆祝动画（五彩纸屑效果）+排名展示（你在第X名），可重新开始
  - 整体界面采用中式卷轴/竹简装饰风格

#### 诗词排行榜
- 诗词比赛结束后显示今日排行榜（本地预设数据），包含：
  - 前5名玩家名称（中式风格名字）+分数
  - 玩家自己的名次和分数高亮显示
  - 底部显示"再来一局"按钮

### 2.5 活动页：婚礼宴请（多人场景）

#### 婚礼场景
- 在活动页中增加一个"婚礼宴请"入口卡片，显示"进行中"标签+"8位宾客在场"
- 点击后进入一个中式宴会厅场景（等距视角场景：红色灯笼悬挂、4-6张圆桌、红绸装饰、中央有拜堂花堂）
- 场景中的"宾客"混合了模拟玩家和真实玩家：
  - 6-8个模拟玩家宾客坐在各自座位上，偶尔会站起来走动或弹出聊天气泡（如"恭喜恭喜"、"百年好合"等祝福语）
  - 如果有其他真实玩家同时进入婚礼场景，他们的角色也会显示在场景中
- 玩家角色可以在宴会厅内走动
- 互动功能：
  - 点击"送红包"按钮，弹出红包动画飞向中央花堂方向，显示"祝福送上！"+全场宾客同时弹出"恭喜"气泡（营造众人齐贺的社交氛围）
  - 点击"放烟花"按钮，场景顶部出现烟花粒子动画，所有在场宾客头顶弹出"哇"表情
  - 点击附近的宾客可以发送一句预设祝福语（如"新婚快乐"、"百年好合"），该宾客会回复一个笑脸表情

### 2.6 集市页：虚拟商店 + 社交动态

#### 商品展示
- 展示一个虚拟商品列表（至少8件商品），采用中式卡片式布局：
  - 商品用彩色方块+文字名称表示（如"汉服·青衫"、"发簪·步摇"、"折扇·山水"、"灯笼·流光"、"玉佩·如意"、"书卷·竹简"等）
  - 每件商品显示名称、价格（用"灵石"虚拟货币）、一个"热门"或"新品"标签
  - 点击商品弹出详情弹窗，显示商品大图（彩色方块放大）+描述+价格+"购买"按钮
  - 用户初始拥有500灵石
  - 点击购买后：如果余额足够，扣款并显示"购买成功！"提示+绿色勾动画；余额不足显示"灵石不足"红色提示
  - 已购买的商品在列表上显示"已拥有"标签

#### 社交动态滚动条
- 集市页底部或侧边显示一个"坊间动态"滚动信息条，模拟其他玩家的活动：
  - "云游书生 购买了 汉服·青衫"
  - "青莲居士 在诗词飞花令中获得第一名"
  - "月下花影 加入了婚礼宴请"
  - 消息每隔3-5秒滚动出现一条（从预设的20条中随机取），营造"这个世界很热闹"的社交氛围
  - 真实玩家的购买行为也会出现在动态中（如果有WebSocket连接）

### 2.7 "我"页面：个人装扮 + 社交名片

#### 角色展示
- 顶部显示角色当前形象（大图）+名称+灵石余额
- 下方显示已购买的服饰/配饰列表（网格布局）
- 点击已购买的服饰可以"穿戴"（角色形象立即更新对应的颜色/装饰变化，简单用颜色块或CSS变化表示即可）

#### 社交名片
- 角色信息下方显示一个"社交名片"卡片，包含：
  - "今日步数"（累计移动距离，本地统计）
  - "参与活动"次数
  - "拥有物品"数量
  - "社交互动"次数（聊天、送红包、送祝福等累计）
- 有一个"分享截图"按钮（点击后用canvas.toDataURL生成当前页面截图并弹出下载提示）

## 三、视觉设计要求（美观度/设计体验——占20%）

### 3.1 整体美术风格
- **中式奇幻风格**：融合中国传统美学元素与奇幻色彩
- **配色方案**：
  - 主背景：深靛蓝 #0a0e1a → 深紫蓝 #1a1a3e 渐变
  - 强调色：琥珀金 #f59e0b（用于按钮、高亮、重要信息）
  - 辅助色：青碧 #06b6d4（用于次要高亮、标签）
  - 文字色：浅灰白 #e8ecf4
  - 卡片/面板：半透明深蓝 rgba(17,24,39,0.85) + 细边框
  - 模拟玩家颜色：使用柔和的区分色（淡紫 #a78bfa、浅绿 #6ee7b7、暖粉 #fbbf24等），与玩家角色的琥珀金色形成区分
- **字体**：中文使用系统默认的思源黑体/苹方，标题可用衬线体模拟书法感
- **装饰元素**：中式祥云纹（用CSS/SVG简单绘制）、竹简纹理、水墨晕染效果

### 3.2 等距视角场景美术
- 山体用多层等距梯形叠加，填充绿色→灰白色渐变模拟雪山顶
- 建筑用简化的等距方块+三角形屋顶表示，填充暖木色 #8B4513 + 红色柱子
- 树木用等距圆形+树干表示，樱花树用粉白渐变圆
- 水面/瀑布用半透明蓝色+波纹动画
- 云雾用多个半透明白色圆形缓慢移动
- 所有角色（玩家+模拟玩家+NPC）用等距视角的小人形表示，通过颜色区分身份

### 3.3 UI组件设计
- 按钮采用圆角矩形，hover时有微妙的发光效果（box-shadow），点击时有缩放反馈
- 卡片采用玻璃拟态（glassmorphism）：半透明背景+模糊+细边框
- 对话框/弹窗有淡入+轻微上浮的入场动画
- 导航栏有磨砂玻璃效果
- 聊天气泡采用圆角白色气泡+小三角指向，有弹入动画
- 在线人数指示器用绿色小圆点+"X人在线"文字
- 所有文字对比度要高（WCAG AA标准），在深色背景上清晰可读

### 3.4 动画与动效（至少包含以下动效）
- 角色走路时的2帧动画（左右脚切换）
- 模拟玩家自主走动+走路动画
- 樱花花瓣飘落粒子效果（至少10个粒子同时飘落）
- 昼夜循环的天空颜色渐变（60秒一周期）
- 云雾缓慢飘动
- 聊天气泡弹入+3秒后淡出动画
- 诗词答对时的绿色光圈扩散+文字弹出
- 烟花粒子爆炸效果（婚礼场景）
- 购买成功时的对勾弹出动画
- Tab切换时的滑动过渡
- NPC头顶对话气泡的弹入动画
- 红包飞行动画（婚礼场景）
- 坊间动态滚动条文字滑入效果
- 模拟玩家之间靠近时的挥手表情动画

## 四、交互设计要求（交互友好度——占20%）

- **移动端优先**：所有交互元素大小不小于44px，适配手机屏幕
- **键盘+触屏双支持**：WASD/方向键移动角色，同时支持点击/触摸地面移动
- **聊天系统**：按Enter键或点击底部聊天输入框激活聊天，输入文字后按Enter发送，消息以气泡显示在角色头顶
- **操作引导**：首次进入时在屏幕上方显示提示"点击地面移动 | 按Enter聊天 | 探索奇幻世界"，5秒后自动消失
- **社交提示**：当模拟玩家发送聊天时，屏幕边缘有微弱的提示动画引导用户注意
- **状态反馈**：所有按钮点击后有明确的视觉/动画反馈，不能有无响应的操作
- **信息层次**：重要信息（灵石余额、在线人数、当前得分）始终可见，次要信息折叠或延迟加载
- **错误处理**：余额不足等错误用友好的内联提示，不使用alert
- **多人感知**：当新的真实玩家加入时，场景中显示一个"XXX 加入了世界"的系统提示（类似游戏入服提示）

## 五、技术实现要求

### 5.1 前端（主要交付物）
- 单个自包含HTML文件，所有CSS写在`<style>`标签内，所有JS写在`<script>`标签内
- 使用Canvas 2D API渲染等距视角场景和所有角色（玩家+模拟玩家+NPC）
- 不依赖任何外部CDN
- 所有数据（商品列表、诗词题目、NPC对话、模拟玩家名字、聊天内容等）以JavaScript对象/数组形式内嵌在代码中
- 页面加载后3秒内可交互
- 在Chrome/Safari最新版本中正常运行
- 响应式设计，在375px-1920px宽度范围内均可正常显示和操作
- 代码结构清晰，关键函数有中文注释

### 5.2 多人同步服务（附带的极简WebSocket服务）

在HTML文件中以注释形式附带一段Node.js WebSocket服务代码（约80行），评委可选择性运行来实现真实多人同步：

```
附：server.js（可选运行，用于真实多人同步）
- 使用Node.js原生WebSocket（无需安装任何npm包）
- 仅做三件事：1) 广播玩家位置 2) 广播聊天消息 3) 广播玩家加入/离开
- 前端通过 try { new WebSocket('ws://localhost:8080') } 连接，连接失败时自动回退到纯模拟玩家模式
- 服务端代码以HTML注释形式内嵌在文件末尾，附运行说明：node server.js
```

前端逻辑：
1. 页面加载后尝试连接 ws://localhost:8080
2. 连接成功：发送自己的位置和名称，接收其他玩家位置并渲染，进入"真实多人模式"
3. 连接失败（服务端未启动）：自动启用模拟玩家模式，单人体验完整可用
4. 两种模式对用户透明，用户无需关心是哪种模式

### 5.3 模拟玩家AI逻辑
- 每个模拟玩家维护：名称、颜色、当前位置、目标位置、状态（走动/停留/聊天）
- 每帧更新：朝目标移动，到达后随机停留2-5秒，选择新目标点
- 每10-20秒随机触发一次聊天气泡
- 模拟玩家数量根据真实玩家数量动态调整：max(1, 5 - 真实玩家数)

## 六、社交感设计要点总结

本Demo的社交感通过三层递进实现：

1. **第一层：世界有人气**（模拟玩家自主走动+聊天+坊间动态滚动）——单人打开就感受到"这里很热闹"
2. **第二层：活动有同伴**（诗词比赛参与者列表+排行榜+婚礼宾客在场）——参与活动时感受到"有人一起"
3. **第三层：真实可互动**（WebSocket实时同步+文字聊天）——开两个浏览器验证"这是真社交"

评委演示建议：先单浏览器展示三层社交感，再开第二个浏览器展示实时同步，社交卖点完整闭环。
